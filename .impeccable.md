# Impeccable Context

本文件是项目的长期设计规范，供前端设计类任务直接读取和执行。

- 它定义这个产品的视觉系统、组件规格和文案规则。
- 它不是 PRD、技术设计或实现 tracker 的替代品。
- 产品定位或品牌方向变化时需同步更新。
- 新的视觉决策必须在这里沉淀为可执行规则，不要依赖会话截图长期存在。

---

## 1. 产品定位

AnchorDesk 是私有资料库知识工作台。用户围绕明确主题（项目、客户、研究方向）组织资料、问答、沉淀结论。

核心气质：**可信、克制、安静**。它是工具，不是玩具。

---

## 2. 色彩系统

项目使用暖中性色体系，所有颜色通过 CSS 变量定义在 `globals.css`：

| 语义 | 变量 | 值 | 用途 |
|------|------|------|------|
| 背景 | `--bg` | `#f7f4ee` | 页面大背景 |
| 高亮背景 | `--bg-elevated` | `#fbfaf6` | body 渐变叠加层 |
| 侧栏 | `--sidebar` | `#f4f0e8` | 左侧导航背景 |
| 表面 | `--surface` | `#ffffff` | 卡片、面板 |
| 柔和表面 | `--surface-soft` | `#fcfaf5` | 输入框底、次级面板 |
| 强调表面 | `--surface-strong` | `#eee8dc` | badge 底色、avatar 底色 |
| 主文本 | `--text` | `#171612` | 标题、正文 |
| 次级文本 | `--muted-strong` | `#696354` | 标签、说明 |
| 弱文本 | `--muted` | `#8b8477` | placeholder、辅助 |
| 品牌色 | `--accent` | `#9b7047` | eyebrow、链接、品牌标记 |
| 主操作色 | `--primary` | `#1a1814` | 按钮、logo 底 |
| 反白 | `--primary-contrast` | `#fffaf3` | 主色上的文字 |
| 边框 | `--border` | `#e7e1d6` | 容器边框 |
| 强边框 | `--border-strong` | `#ddd5c8` | hover、focus 边框 |

**规则**：
- 不引入纯白背景、冷蓝灰色、高饱和品牌色。
- 强调色（accent）仅用于 eyebrow 标签、小面积链接、品牌标记，不大面积铺色。
- 红色系仅限危险操作：`text-red-600`、`bg-red-50`、`border-red-200`。
- 绿色系仅限成功状态：`text-emerald-800`、`bg-emerald-50`、`border-emerald-200`。

## 3. 字体与排版

```css
--font-sans: "PingFang SC", "Hiragino Sans GB", "Source Han Sans SC", "Microsoft YaHei", sans-serif;
--font-serif: "Iowan Old Style", "Baskerville", "Songti SC", "Noto Serif SC", serif;
```

- **sans-serif 为主**。所有 UI 文本、按钮、标签、正文用 sans。
- **serif 仅限品牌标题**：产品名 `AnchorDesk`、页面主 heading 中偶尔使用。
- h1/h2/h3 默认 `font-family: var(--font-serif); letter-spacing: -0.03em`（globals.css 已设置）。

**字号层级**：

| 用途 | 字号 | 字重 | 示例 |
|------|------|------|------|
| 页面主标题 | `text-[1.5rem]` ~ `text-[1.85rem]` | `font-semibold` | 工作空间名 |
| 面板标题 | `text-[15px]` | `font-semibold` | 弹出面板、会话标题 |
| 正文/按钮 | `text-sm` / `text-[14px]` | `font-medium` | 菜单项、按钮文字 |
| 元数据/键值行 | `text-[13px]` | 正常 / `font-medium` | 创建者、时间 |
| 辅助/badge | `text-[12px]` ~ `text-[11px]` | `font-medium` / `font-semibold` | chip、eyebrow |
| Eyebrow | `text-[11px]` | `font-semibold uppercase tracking-[0.14em]` | 区域标签 |

**文案规则**：
- 短文案不加句号。
- 不写"欢迎语、功能复述、路径解释"等废话。
- 标题、按钮已经表达清楚的，不再补一行重复。
- 删掉后不影响操作和理解的文案，就不应出现。

---

## 4. 阴影

```css
--shadow-soft: 0 12px 28px rgba(23, 22, 18, 0.04);
--shadow-card: 0 20px 44px rgba(23, 22, 18, 0.06);
```

- `shadow-soft`：卡片、sidebar 选中态、轻浮层。
- `shadow-card`：弹出面板、下拉菜单、dialog。
- 不使用强阴影、多层阴影或发光效果。

## 5. 圆角体系

| 用途 | 圆角 |
|------|------|
| 页面级面板 | `rounded-[28px]` ~ `rounded-[32px]` |
| 卡片/tile | `rounded-[24px]` ~ `rounded-2xl` |
| 弹出面板/popover | `rounded-2xl`（16px） |
| 菜单项/按钮 | `rounded-xl`（12px） |
| 输入框 | `rounded-[18px]` ~ `rounded-2xl` |
| pill 按钮 | `rounded-full` |
| chip/badge | `rounded-full` |
| Logo container | `rounded-xl` |
| Avatar | `rounded-full` |

**规则**：弹出面板不使用 `rounded-[26px]`、`rounded-[28px]` 等过度圆角。

---

## 6. 间距节奏

- 模块间：`gap-6` ~ `gap-8`
- 面板内分组间：`gap-4` ~ `gap-5`
- 紧凑列表/行间：`gap-0.5` ~ `gap-2`
- 弹出面板内部：`p-1.5`，分区间 `my-1.5`
- 页面 padding：`px-6 py-8 md:px-8`

避免 `gap-10` 以上的大间距。内容靠上组织，操作贴近内容。

## 7. 组件规格

### 7.1 按钮

使用 `buttonStyles()` 函数统一生成。

| variant | 外观 | 场景 |
|---------|------|------|
| `primary` | 深色实底 `bg-app-primary text-app-primary-contrast` | 主操作 |
| `secondary` | 浅底细边框 `bg-white/90 border-app-border` | 次级操作 |
| `ghost` | 透明底 `bg-transparent text-app-muted-strong` | 导航链接、工具栏次级 |
| `danger` | 红底白字 | 危险确认 |
| `dangerGhost` | 透明底红字 | 危险操作（弱） |

| size | 场景 |
|------|------|
| `md` | 独立表单提交 |
| `sm` | 工具栏、面板内 |
| `xs` | 弹出面板内、紧凑操作行 |

**规则**：弹出面板内只用 `xs` 或 `sm`。页面顶部工具栏次级操作用 `ghost`，主操作用 `primary`。

### 7.2 弹出面板（Popover / Dropdown）

所有弹出面板（省略号菜单、分享面板、用户菜单、下拉选择）统一规格：

```
容器：rounded-2xl border border-app-border bg-white/98 p-1.5 shadow-card backdrop-blur-md
宽度：w-[min(320px,calc(100vw-24px))]
```

内部结构：
- **区头**：`px-3 pt-2.5 pb-1`，eyebrow 用 `text-app-accent`，标题用 `text-[15px] font-semibold`
- **分割线**：`mx-2 my-1.5 h-px bg-app-border/70`（不用 `border-t` + `pt-4`）
- **键值行**：`px-3 py-0.5 text-[13px]`，label `text-app-muted-strong`，value `font-medium text-app-text`
- **chip**：`text-[11px] px-2.5 py-0.5 rounded-full border border-app-border bg-white/86`
- **操作行**：`rounded-xl px-3 py-2` + `menuItemStyles()`，图标占位 `size-7`
- 不嵌套子卡片。不写冗余描述文案。

### 7.3 菜单项

使用 `menuItemStyles()` 函数。

- 默认态：`text-app-muted-strong hover:bg-app-surface-soft/82 hover:text-app-text`
- 选中态：`bg-app-surface-soft font-medium text-app-text`
- 危险态：`text-red-600 hover:bg-red-50`

结构：`flex items-center gap-2.5 rounded-xl px-3 py-2`，图标用 `size-7` 占位容器包裹。

### 7.4 输入框

使用 `inputStyles()` 函数。

- md：`h-12 rounded-2xl px-4 text-sm`
- compact：`h-10 rounded-[18px] px-3.5 text-[14px]`
- 底色：`bg-app-surface-soft border border-app-border`
- focus：`border-app-border-strong ring-app-accent/10`

### 7.5 卡片/Tile

使用 `workspaceTileStyles()` 或 `ui.panel`。

- panel：`rounded-[28px] border border-app-border bg-white/90 p-6 shadow-soft`
- tile：`rounded-[24px]` + hover `translate-y-0.5`
- 卡片底部统计区用 `border-t border-app-border pt-4 text-sm text-app-muted`

### 7.6 页面头部

- **工作台内页**：面包屑 + 右侧 topActions，轻薄工具栏。
- **首页/列表页**：品牌 logo + 标题 + 右侧操作按钮，次级操作用 `ghost`，主操作用 `primary`。
- **公开页面**（share 等）：sticky header `border-b border-app-border/60 bg-app-bg-elevated/80 backdrop-blur-md`，品牌 logo + 页面类型标识。

### 7.7 图标

- 统一使用 `IconBase` 组件包裹 SVG path。
- 默认 `viewBox="0 0 20 20"`，`stroke="currentColor"`，`fill="none"`。
- 笔画宽度 `1.6` ~ `1.8`。
- 默认尺寸 `size-4`（16px），弹出面板内操作图标 `size-[18px]`。
- 始终加 `shrink-0`。

## 8. 页面级布局

### 侧栏

- 宽度：`258px`（桌面端 `min-[720px]`）
- 背景：`bg-app-sidebar`
- 结构：品牌区 → 新建会话按钮 → 历史列表 → 底部导航 → 用户面板
- 项目间距紧凑，字号偏小（`text-[13px]`）

### 内容区

- `grid-rows-[auto_minmax(0,1fr)]`
- 顶部 header 用 `border-b border-app-border` 分隔
- 内容列宽度约束：对话区 `max-w-[1080px]`，表单/设置 `max-w-[1040px]`

### 响应式

- `min-[720px]`：切换为侧栏 + 内容布局
- 移动端：drawer overlay + hamburger 按钮

---

## 9. 不做的事

- 不做霓虹、赛博、渐变发光、深色面板。
- 不做消费化运营元素（积分卡、勋章、活动位）。
- 不做大气泡、大头像聊天界面。
- 不做极端稀疏的展示型布局。
- 不在弹出面板中嵌套带圆角和阴影的子卡片。
- 不写"解释面板用途"的冗余帮助文案。
- 不在同一行放多个同级别实底按钮竞争视觉焦点。
- 不使用 TailwindCSS 默认蓝色系颜色。

---

## 10. 设计决策原则

1. **可测量**：本文件的规则应具体到 class name 或数值，不写"适当留白""合理圆角"。
2. **单一来源**：颜色、间距、圆角的规格以本文件和 `lib/ui.ts` + `globals.css` 为准。
3. **减法优先**：字段、说明、装饰、交互先压到最小集合。
4. **一屏一主任务**：每个页面一个清晰主目标，不做多个同级 CTA 竞争。
5. **规则可沉淀**：新的视觉偏好必须提炼为这里的明确规则，不停留在"像某产品"。
