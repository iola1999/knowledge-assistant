# Impeccable Context

本文件用于沉淀项目的长期设计上下文，供后续前端设计类任务直接读取。

- 它回答的是“这个产品应该给人什么感觉、服务谁、设计时优先什么”。
- 它不是 PRD、技术设计或实现 tracker 的替代品。
- 当产品定位或品牌方向发生明显变化时，需要同步更新这里。
- 评审过的截图、临时参考图和口头偏好，如果会影响后续视觉方向，必须在这里沉淀成可执行规则；不要依赖会话里的图片长期存在。

## Design Context

### Users
- 个人知识工作者、小团队负责人、产品/运营/咨询等需要长期整理资料的人。
- 他们通常在桌面端围绕一个明确主题工作，例如项目、客户、产品线、研究方向或阶段性任务。
- 他们使用这个产品的核心目标不是“和 AI 随便聊聊”，而是围绕私有资料库完成可信问答、整理结论、沉淀会话和导出结果。
- 他们最看重的是依据可追溯、信息隔离清晰、流程稳定，而不是花哨功能。

### Brand Personality
- 品牌气质应长期保持为：可信、克制、清晰。
- 界面需要传达的感受是：冷静、可靠、专注，而不是兴奋、炫技、夸张。
- 它是“知识工作台”而不是“玩具式 AI 聊天应用”。

### Aesthetic Direction
- 默认采用浅色、暖中性色、文档感较强的视觉方向，接近纸面、书页、案头工作区，而不是科技炫光风格。
- 界面要优先服务阅读、检索、写作和引用跳转，强调信息层次和可扫描性。
- 页面应避免过度留白造成“空”，也避免堆叠卡片造成“闷”；要让内容密度服务任务完成。
- 主要视觉语义应围绕“空间、资料、证据、工作流”展开，而不是围绕抽象 AI 意象展开。
- 交互动效应低噪、短促、功能性优先；不依赖炫技动画建立品质感。
- 默认中文阅读场景优先，文案和版式都应照顾中文信息密度。

### Interface Visual Language
- 整体画面应接近高质量桌面知识工作台：大面积温和浅底，低饱和暖灰侧栏，白色或奶白色内容面板，几乎没有强烈品牌色块。
- 布局优先采用“稳定左侧导航 + 居中主工作区”的结构。左栏应像资料目录和历史入口，安静、恒定、低干扰。
- 主内容区默认使用窄于整屏的阅读列。对话、结果、状态卡、表单都应围绕清晰主列展开，而不是无限铺满宽屏。
- 顶部工具栏要轻，不做厚重导航条。标题、模式切换、次级操作使用小尺寸图标和轻量标签组织。
- 输入区、状态卡、过程块、结果附件块都采用轻边框、轻阴影、柔和圆角的容器体系，强调秩序而不是存在感。
- 用户消息或上下文提示可使用浅暖色气泡，但只能比背景略深一层，不能变成高饱和聊天气泡界面。
- 整体风格必须让人首先感受到“安静、可信、可专注”，其次才是“这是个 AI 产品”。

### Layout And Density
- 页面留白要有节制。允许呼吸感，但不能让内容像漂浮在空白画布中央；中部大面积无信息区域应尽量避免。
- 核心内容列在桌面端应保持明确宽度约束，优先保证长文本阅读、状态理解和输入操作的连续性。
- 表单、卡片和状态模块之间的垂直间距应偏紧凑，常规使用 12/16/20/24 的节奏，避免动辄 32 或 40 以上的空隙。
- 左侧导航中的分组、历史列表和快捷入口应采用小字号、高密度但不拥挤的排列，强调检索效率。
- 创建页、设置页、对话页都应遵循“内容靠上组织，操作贴近内容，避免主按钮漂在很远的底部”。

### Components And Controls
- 输入框、搜索框、对话框和筛选条采用圆角矩形，但高度应克制，视觉上偏薄而稳，不做“充气型”大控件。
- 主按钮允许深色实底，但尺寸应紧凑、结实、可点击，不应出现过宽、过高、像横幅一样压住页面的按钮。
- 次级按钮、标签和 chip 更适合浅底细边框或极低对比底色，避免所有操作都做成同级实体按钮。
- 图标应统一为小尺寸线性图标，笔画轻，尽量与文字形成一体化排版，不依赖图标本身抢视觉焦点。
- 过程状态卡适合采用“多行轻分隔列表 + 展开查看细节”的形式，视觉上像工作记录，而不是营销卡片。
- 文件预览、引用块和结果附件可使用低对比小卡片承载，但必须优先保证标题、来源和状态信息清晰可扫。

### Typography
- 整体以无衬线中文 UI 字体为主，强调干净、可读、稳定。常规工作台页面不要过度依赖装饰性 serif 标题。
- 标题层级应清楚，但差值不宜过大。大多数页面更适合稳重的中等字号标题，而不是夸张 hero 文案。
- 说明文、提示文、状态文案都应短、直白、弱化语气词，像成熟工具的系统语言。
- 文本颜色层级以深文本、次级灰、禁用灰为主，避免彩色文案泛滥。

### Color And Surface Rules
- 背景采用暖白、米白、淡燕麦色一类的中性色体系，整体对比温和，不使用纯白工作台或冰冷蓝灰底。
- 侧栏颜色比主背景略深半级，用于建立层次，但不做重色块。
- 内容容器以白色或近白色为主，边框极浅，阴影稀薄，营造纸面和卡夹感。
- 强调色只作为小面积点缀，例如焦点、链接、少数品牌标记或激活态；不能大面积铺色。
- 主操作色宜接近深棕黑、墨黑、暖黑，而不是纯黑或高饱和品牌色。

### Explicit Non-Goals
- 不做霓虹、赛博、渐变发光、深色酷炫 AI 面板风格。
- 不做消费化运营视觉，例如积分卡、勋章、礼物卡、悬浮宠物、活动位。
- 不做社交聊天产品式的大气泡、大头像、大彩色标签体系。
- 不做极端稀疏的展示型布局。这个产品是工作工具，不是品牌海报。
- 不照搬任何外部产品的功能架构、命名或交互结构，只保留适合本产品的视觉秩序和控件尺度。

### Design Principles
- 证据优先：任何核心界面都应优先帮助用户理解资料、引用和结论之间的关系。
- 一屏一主任务：每个页面必须有一个清晰主目标，避免多个同级 CTA 互相竞争。
- 先减法再加法：字段、说明、装饰和交互都应先压到必要最小集合，再考虑扩展。
- 助手服务工作流：AI 是完成知识工作的手段，不是页面视觉中心本身。
- 温和但不松散：保持暖色、安静、可信的气质，同时避免过大间距、过胖控件和低效布局。
- 设计规则可复用：以后新的截图或视觉偏好，也必须继续被提炼成这里的明确规则，而不是停留在“像某某产品”的描述层。
