## 1. 产品定位

构建一个面向个人用户和小团队的工作空间级 AI 助手。系统以 `Workspace` 作为会话上下文，以“工作空间私有资料库 + 可订阅全局资料库”作为知识边界，必要时联网补充公开信息；所有关键回答默认要求可追溯、可点击、可核验。

产品原则：

- 默认“基于证据回答”，不是“基于记忆回答”。
- 默认“工作空间上下文驱动 + library scope 授权”，不是把所有资料混成全局共享上下文。
- 默认“先检索再生成”，不是直接自由发挥。
- 默认“先给依据、再给结论”，不是只输出结果。
- 默认“支持人工接管”，不是追求全自动闭环。

非目标：

- 不做团队协作、审批流、组织治理和复杂权限体系。
- 不做平台内富文本编辑器。
- 不在证据不足时输出肯定性结论。

## 2. 目标用户

- 个人知识工作者：围绕项目、客户、主题资料持续问答和总结。
- 产品/运营/咨询角色：围绕说明文档、会议纪要、方案资料建立私有知识库。
- 小团队负责人：围绕固定主题沉淀资料并生成阶段性报告。

## 3. 核心场景

### 3.1 工作空间问答

用户在某个工作空间中提问。系统优先检索该 workspace 当前可访问的资料范围，默认包括该 workspace 的私有资料库和已启用的全局资料库订阅，返回带引用的回答，并支持点击引用跳回原始文档页码。用户可按会话选择一个已启用的 Claude-compatible 模型；该选择会按 conversation 持久化。

### 3.2 外部公开信息补充

当本地资料不足时，系统补充通用网络搜索结果，并在回答中区分“工作空间资料依据”和“外部公开来源”。

### 3.3 长文结果生成

用户要求生成摘要、整理稿、调研提纲、阶段性报告或执行建议。系统可先生成大纲，再逐段写作，最后支持导出。

### 3.4 资料上传与消化

用户上传 PDF、DOCX、TXT、Markdown 等文本型资料。系统异步执行文本抽取、结构解析、切块、向量化和索引，并展示处理状态。图片、扫描件和纯 OCR 材料当前不在 P0 可用范围。

补充：用户在首条消息前还可上传“会话级临时资料”；这些附件只在当前会话内提供 parse-only 检索、按页读取和引用能力，不进入 workspace 全局检索。

### 3.5 专项法规检索

在通用工具之外，系统保留一个专项的“法律条文搜索”工具，用于需要法规或法条引用的场景。它不是产品主定位，但作为专门工具保留。

## 4. 信息架构

- `User`：个人账号。
- `Workspace`：会话上下文、报告容器和授权入口。
- `Knowledge Library`：资料归属、目录组织、检索过滤和引用来源的主边界。
- `Workspace Library Subscription`：workspace 对全局资料库的挂载、检索与只读访问关系。
- `LLM Model Profile`：super admin 维护的 Claude-compatible 模型配置，供对话与报告链路复用。

原则：

- 聊天和报告以 `Workspace` 为一等对象。
- 文档、目录、检索过滤和引用来源以 `Knowledge Library` 为一等对象。
- 每个 `Workspace` 自动拥有一个 private library，并可订阅多个 global library。
- 一个用户可创建多个工作空间。
- 引用必须落到具体文档版本，而不是模糊地指向文件名。

## 5. 功能需求

### 5.1 账号与基础权限

P0：

- 用户名密码注册登录。
- 所有资源按用户和工作空间隔离。
- 支持修改密码、退出登录。

P1：

- 邮箱绑定或密码找回。
- 更细粒度的个人偏好。

### 5.2 工作空间与资料管理

P0：

- 创建、删除工作空间。
- 删除为软删除；删除后从默认列表隐藏，当前不提供恢复入口。
- 每个工作空间自动拥有一个私有资料库。
- 工作空间可配置统一提示词，对该空间内所有会话默认生效。
- 资料按逻辑目录组织、移动、重命名、删除。
- workspace owner 可订阅、暂停或移除全局资料库；已订阅全局资料在工作空间资料页中以只读挂载方式出现。
- 支持文档类型和标签。
- 支持版本管理和重复文件识别。

P1：

- 工作空间模板。
- 多工作空间联合检索。

### 5.3 上传与异步处理

P0：

- 当前阶段支持 PDF、DOCX、TXT、Markdown。
- 图片、扫描件和纯 OCR 材料待商业 OCR provider 方案确认后再纳入。
- 上传后立即返回任务状态，不阻塞界面。
- 展示 `queued / extracting_text / parsing_layout / chunking / embedding / indexing / ready / failed`。
- 失败任务必须展示失败阶段和建议。
- 对相同文件内容复用解析缓存。
- 保存页码、段落、标题层级、表格和坐标等结构化信息。

P1：

- 低质量解析告警。
- 人工校正入口。

### 5.4 对话与 Agent 能力

P0：

- 在工作空间内发起多轮对话。
- 会话支持选择已启用的 Claude-compatible 模型，选择按 conversation 维度持久化。
- 模型回答以流式输出呈现。
- 展示工具调用状态，不展示思维链。
- 支持在回答 streaming 期间主动停止当前生成，并保留已生成片段作为这一轮的结果。
- 支持在首条消息前上传会话级临时资料；这些附件在当前会话内可被检索、按页读取和引用，但不进入 workspace 全局检索。
- 支持为单个会话生成可撤销的公开只读分享链接；公开页不暴露空间内资料跳转。
- 当最新 assistant 回答失败时，支持基于上一条 user prompt 重新生成当前回答。
- 支持以下工具：
  - 工作空间资料检索
  - 会话级临时资料检索与页窗读取
  - 法律条文搜索
  - 通用网络搜索
  - 文档片段读取
  - 网页/PDF 内容抓取
  - 报告大纲生成
  - 报告分段写作
  - 导出 DOCX
- 回答默认采用“本地资料优先 + 联网查询补充”，不提供模式开关。
- 支持严格引用模式：无引用不输出肯定结论。

P1：

- 分支对话。
- 更多结构化研究工具。

### 5.5 报告与导出

P0：

- 支持摘要、说明稿、阶段报告、调研结果整理。
- 平台内不做在线编辑器，重点是 Agent 生成结果、引用和导出。
- 导出 DOCX。

P1：

- 导出 PDF。
- 自定义模板。

### 5.6 阅读与引用体验

P0：

- 右侧 PDF/文档阅读器。
- 支持页码、搜索、缩放、文本选择。
- 点击回答中的引用后，自动定位到页码和段落。

P1：

- 对比阅读。
- 摘录收藏和批注。

### 5.7 搜索与过滤

P0：

- 支持语义检索和关键词检索混合。
- 支持按工作空间、文档类型、标签、文件名过滤。

P1：

- 更细粒度排序策略和过滤器。

### 5.8 可控性与防幻觉

P0：

- 检索不足时明确说“当前资料不足以支持结论”。
- 回答区分 `直接依据 / 基于依据的推理 / 待补充信息`。
- 回答中应展示引用摘录与来源归属，明确区分工作空间资料、全局资料库和外部公开来源。
- 无法完成引用校验时，不应把该轮回答当作成功结果落成。
- 禁止伪造引用。

P1：

- 回答后二次核验。
- 自动证据反查。
