# 律师行业 AI 助手 PRD

版本：v0.1  
日期：2026-03-28

## 1. 产品定位

构建一个面向律师、法务和律师助理的案卷级 AI 工作台。系统以用户私有资料库为主，按需联网补充公开信息，并且所有关键结论默认要求可追溯、可点击、可核验。

产品原则：

- 默认“基于证据回答”，不是“基于记忆回答”。
- 默认“案卷/项目级隔离”，不是“全局共享上下文”。
- 默认“先检索再生成”，不是直接让模型自由发挥。
- 默认“先给依据、再给观点”，不是只给结论。
- 默认“支持人工接管”，不是追求全自动闭环。

非目标：

- 不承诺替代律师独立执业判断。
- 不在无依据时输出肯定性法律结论。
- 不把未经授权的公开网站长期抓取入库。

## 2. 目标用户

- 执业律师：围绕具体案件做法规检索、证据梳理、报告起草、合同审查。
- 公司法务：围绕合同、制度、合规材料建立私有知识库并持续问答。
- 律师助理：做资料预处理、事实摘要、时间线整理、初稿撰写。
- 团队管理员：管理成员、权限、配额、审计日志、模型与工具策略。

## 3. 核心使用场景

### 3.1 案卷问答

用户在某个案卷中提问，例如“结合当前卷宗，判断对方关于不可抗力免责的主张是否成立”。系统优先检索该案卷知识库，返回带引用的回答，并支持点击引用跳转到 PDF 原文定位。

### 3.2 法规/案例补充检索

当本地资料不足，且用户允许联网时，系统优先检索官方或已授权法律来源，再补充通用网络结果。回答中必须区分“案卷材料依据”和“外部公开来源”。

### 3.3 合同/文书审查

用户选择一份合同、证据清单或法律意见初稿，使用预设动作快速生成风险点表、缺失条款、争议焦点、证据矛盾点。

### 3.4 报告撰写

用户要求生成法律意见书、尽调报告、案件事实摘要、诉讼策略备忘录。系统先生成大纲，再逐段写作，过程中允许用户编辑和继续生成。

### 3.5 资料上传与消化

用户上传 PDF、DOCX、图片等资料到知识库。系统自动排队执行 OCR、版面解析、语义切块、向量化与入库，并实时展示处理状态和失败原因。

## 4. 信息架构

推荐采用四层对象模型：

- `Organization`：律所/团队租户。
- `Workspace`：业务空间，例如团队、客户线、部门。
- `Case`：案卷/项目，是知识隔离和对话上下文的核心单位。
- `Knowledge Base`：挂在案卷或工作空间下的资料集合。

原则：

- 聊天、资料、报告、标签、权限都以 `Case` 为一等对象。
- 一个用户可访问多个工作空间和案卷，但默认只能看授权范围内的数据。
- 引用跳转必须回到具体文档版本，而不是模糊地指向“某份文件”。

## 5. 功能需求

### 5.1 账号、组织与权限

P0：

- 邮箱/手机号注册登录。
- 支持组织、成员、角色。
- 支持基础 RBAC：管理员、律师、助理、只读访客。
- 所有资源具备租户隔离和案卷级访问控制。

P1：

- 企业 SSO（OIDC/SAML）。
- 外部客户只读共享。
- 细粒度权限：是否允许联网、是否允许导出、是否允许查看原文。

### 5.2 案卷与知识库管理

P0：

- 创建、归档、删除案卷。
- 资料按案卷上传、移动、删除、重命名、标签化。
- 支持文档类型标签：合同、起诉状、答辩状、证据、法律法规、案例、邮件、会议纪要。
- 支持版本管理和重复文件识别。

P1：

- 案卷模板。
- 多案卷联合检索。
- 案卷间共享资料但保留独立权限。

### 5.3 上传与异步处理

P0：

- 支持 PDF、DOCX、TXT、Markdown、图片；P1 支持压缩包批量导入、EML、XLSX。
- 上传后立即返回任务状态，不阻塞用户界面。
- 展示细粒度状态：
  - `queued`
  - `extracting_text`
  - `parsing_layout`
  - `chunking`
  - `embedding`
  - `indexing`
  - `ready`
  - `failed`
- 失败任务必须展示失败阶段和可操作建议。
- 对扫描件自动触发 OCR。
- 保存页码、段落、标题层级、表格、版面坐标等结构化信息。

P1：

- 低质量解析告警。
- 人工校正入口。
- 用户手动补充标签、文档类型、关键当事人、时间范围。

### 5.4 对话与 Agent 能力

P0：

- 在案卷内发起多轮对话。
- 模型回答以流式输出呈现。
- 展示工具调用状态，不展示原始思维链。
- 支持以下工具：
  - 私有知识库检索
  - 文档片段读取
  - 公开网络搜索
  - 网页/PDF 内容抓取
  - 报告大纲生成
  - 报告分段写作
  - 导出 DOCX/PDF
- 每条回答支持点击引用，右侧打开原始文档并定位高亮。
- 支持“仅资料库回答”和“允许联网”两种模式切换。
- 支持“严格引用模式”：无引用不输出结论。

P1：

- 分支对话。
- 多 Agent 协作，例如“法规检索代理”“证据整理代理”“文书写作代理”。
- 结构化办案工具：事实时间线、争点树、证据链视图、相互矛盾点检测。

### 5.5 报告写作与导出

P0：

- 支持法律意见书、尽调报告、案件摘要、合同风险审查报告。
- 先输出大纲，再确认后分段生成正文。
- 支持边写边改，保留版本历史。
- 导出 DOCX，保留标题层级与引用附录。

P1：

- 导出 PDF。
- 自定义模板与律所抬头。
- 生成引用附录和参考依据清单。

### 5.6 文档阅读与引用体验

P0：

- 右侧 PDF/文档阅读器。
- 支持页码、搜索、缩放、文本选择。
- 点击回答中的引用后，自动定位到具体页和段落。
- 如果文档源自 OCR，仍需尽可能映射到页和块坐标。

P1：

- 对比阅读，两份合同/两版文书差异高亮。
- 证据摘录收藏。
- 用户批注与共享批注。

### 5.7 搜索与过滤

P0：

- 支持语义检索和关键词检索混合。
- 支持按案卷、文档类型、标签、时间范围、文件名过滤。
- 支持搜索结果分组：法规、案例、合同、证据、邮件。

P1：

- 支持按法条编号、合同条款编号、法院层级、案由精确过滤。
- 支持结果排序策略切换：相关度优先、最新优先、证据优先。

### 5.8 可控性与防幻觉

P0：

- 当检索结果不足时明确说“当前资料不足以支持结论”。
- 将回答区分为：
  - `直接依据`
  - `基于依据的推理`
  - `待补充信息`
- 每条回答给出置信提示和依据覆盖度。
- 对无来源结论禁止伪造引用。

P1：

- 回答后二次核验。
- 对关键结论自动执行“证据反查”，验证每条主张是否有支撑片段。

### 5.9 运维与审计

P0：

- 记录上传任务、模型调用、工具调用、导出行为、失败日志。
- 支持配额和成本统计。
- 支持管理员查看审计日志。

P1：

- 质量评测后台。
- A/B 测试不同检索策略和模型组合。

## 6. 关键交互要求

### 6.1 工作台布局

- 左侧：对话区、工具状态、快速动作。
- 右侧：文档阅读器或报告编辑器。
- 移动端保留核心问答和文件查看，复杂编辑以桌面端优先。

### 6.2 快捷动作

聊天输入框支持 `/` 命令：

- `/摘要`：提炼案件事实与争议焦点。
- `/检索`：仅给检索结果，不做结论推理。
- `/风险`：输出合同/文书风险点表格。
- `/比对`：比较两份文档差异。
- `/时间线`：抽取时间、主体、行为、证据来源。

### 6.3 联网优先级

对于需要外部资料的任务，系统应采用固定优先级：

1. 当前案卷资料。
2. 当前工作空间共享资料。
3. 官方法律法规与案例来源。
4. 已授权商业法律数据库。
5. 通用互联网结果。

## 7. 非功能需求

### 7.1 安全

- 租户隔离。
- 静态加密与传输加密。
- 审计日志不可篡改。
- 支持敏感字段脱敏日志。
- 对模型和工具设置显式权限白名单。

### 7.2 性能

- 普通问答首屏流式响应目标 < 5 秒。
- 上传后状态更新延迟目标 < 3 秒。
- 200 页以内普通 PDF 在队列空闲时 2 到 10 分钟内完成处理。
- 长报告允许异步生成并通知。

### 7.3 可用性

- 关键任务支持重试。
- 对话、报告和任务状态具备断点恢复能力。
- 外部搜索不可用时不影响纯案卷问答。

### 7.4 合规

- 支持客户数据不用于模型训练的供应商策略。
- 面向中国法律数据时，外部数据源需区分官方来源、授权来源、普通网页来源。
- 对需私有化部署的客户，架构上保留专有云/VPC 部署路径。

## 8. 成功指标

上线后重点看五类指标：

- 激活：新用户 7 日内至少完成 1 次上传和 3 次有效问答。
- 检索质量：律师标注样本上的 Recall@20、NDCG@10、引用命中率。
- 回答可信度：人工抽检中的无依据结论率、错误引用率。
- 交付效率：生成初稿耗时、人工修改时长下降比例。
- 商业指标：团队留存、单组织活跃案卷数、文档处理量、毛利率。

建议先设定 MVP 验收阈值：

- 检索 Recall@20 >= 0.9（内部样本）。
- 引用命中率 >= 0.95。
- 无依据结论率 <= 2%。
- 上传成功率 >= 98%。

## 9. 分阶段范围

### 9.1 MVP

- 账号登录、组织、案卷。
- 文档上传、异步处理、状态展示。
- 案卷内问答。
- 混合检索 + 重排。
- 可点击引用。
- 大纲先行的报告生成。
- DOCX 导出。
- 基础审计与成本统计。

### 9.2 Beta

- 联网搜索与网页抓取。
- 分支对话。
- 批注与人工纠错。
- 合同比对与时间线。
- 更多文档格式和批量导入。

### 9.3 GA

- 企业 SSO。
- 多组织治理。
- 评测中心。
- 私有化部署。
- 商业法库接入。

## 10. 主要风险

- 法律 PDF 解析质量不足，导致检索上限过低。
- 中文法律术语和案情事实混杂，单纯语义检索容易偏题。
- 如果引用不能精准定位到原文，律师信任建立会失败。
- 联网检索若没有源优先级和域名策略，容易引入低质量依据。
- 报告生成如果不分段、不可编辑，用户很快会回退到 Word 手工写作。

## 11. 待确认问题

- 目标法域是中国法律为主，还是中英双语/多法域？
- 首批客户更偏诉讼、非诉、法务合规还是合同审查？
- 是否需要接入付费法律数据库与案管系统？
- 是否从第一天就要求私有化/VPC 部署？
- 是否需要团队协作能力，例如批注、评论、共享视图？
